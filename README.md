# Portfolio Site

## 概要
Webコーダーとしての自身のスキルと仕事の進め方を伝えるための、オリジナルのポートフォリオサイトです。構成の叩き台づくりにClaude・Codexを活用し、方向性の指示や取捨選択、実装・デバッグ・細部の調整は自身で行いました。静的HTML/SCSS/JavaScriptで制作したのち、独自のWordPressテーマとして移植しています。

## 使用技術
- WordPress（オリジナルテーマ / PHP）
- HTML / SCSS（BEM記法・ファイル分割）
- JavaScript（Vanilla JS）
- カスタム投稿タイプ + ACF（Advanced Custom Fields）によるWorksセクションの動的化

## こだわったポイント
- 長期的な運用を見据え、BEM設計とSCSSのファイル分割を採用
- Worksセクションはカスタム投稿タイプ＋ACFで動的化し、投稿画面から作品追加・更新できる構成に
- header.php / footer.php / front-page.php の「サンドイッチ構造」でテーマを設計
- サイト全体で「思い込みで進めない、確認して正確に」の軸を統一

## つまずいた点・学んだこと
- Works投稿にACFで画像を登録する際、フルページの縦長スクリーンショットをそのままアップロードしてしまい、WordPressの自動リサイズで画質が大きく劣化する問題が発生。表示エリアの比率に合わせて事前にトリミングする形に直した
- header.php内のog:urlなど、静的HTML時代のコードをそのまま移植した箇所にドメイン固定のハードコーディングが残っていた。`home_url()`など動的取得の関数に置き換え、環境が変わっても追従できる形にした
- カスタム投稿タイプ＋ACFでのWorksセクション動的化を実践投入。フィールド設計（12項目）から実データ入力・表示確認までを一通り経験した

## Note
本リポジトリに含まれるのはWordPressテーマ（コード）のみです。Works投稿の実データ（画像・テキスト）はWordPressのデータベース側で管理しているため、このリポジトリには含まれていません。
